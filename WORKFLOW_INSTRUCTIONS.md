# Инструкция по использованию GitHub Actions Workflow

##  Быстрый старт:

### 1. Создайте тег для запуска:
```bash
git tag v1.0-article
git push origin v1.0-article
```
### 2. Ручной запуск:
Перейдите в репозитории на GitHub

Откройте вкладку "Actions"

Выберите "Create Release Artifacts"

Нажмите "Run workflow"

## Что создает workflow:
### Артефакты:
research-artifacts-v1.0-article.zip - архив с результатами

Автоматический релиз на GitHub

## Содержимое архива:
```text
artifacts/
├── results_csv/     # Результаты экспериментов
├── figures/         # Графики и визуализации
├── configs/         # Конфигурационные файлы
└── README.md        # Описание артефактов
```
## Настройка:
Изменение версии Python:

Отредактируйте в .github/workflows/release.yml:

```yaml
env:
  PYTHON_VERSION: '3.10'  # Измените на нужную версию
```

## Добавление тегов для запуска:
```yaml
on:
  push:
    tags:
      - 'v*'           # Все версии
      - 'v1.*'         # Только v1.x
      - 'v2.0-*'       # Конкретная версия
```
## Устранение проблем:
### Workflow не запускается:
Проверьте наличие файла в .github/workflows/

Убедитесь, что есть права на запись

Проверьте синтаксис YAML

### Ошибки при выполнении:
Проверьте логи в GitHub Actions

Убедитесь, что все зависимости установлены

Проверьте пути к файлам в workflow

## Поддержка:
Для вопросов по workflow обращайтесь к maintainers репозитория.
